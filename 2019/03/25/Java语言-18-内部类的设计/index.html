18-内部类的设计

发表于 2019-03-25 | 分类于 做一个有骨气的Coder ， 我的JavaEE哪有这么熟练 ， Java语言 ， 入门篇

内部类概述

内部类（inner class）是定义在另一个类中的类。例如：在类A中定义了一个类B，类B就是内部类。

内部类的特点

  1. 内部类方法可以直接访问外部类（该类定义所在的作用域）中的数据，包括私有的数据。
  2. 外部类要访问内部类的成员，必须创建对象。
  3. 内部类可以对同一个包中的其他类隐藏起来。
  4. 当想要定义一个回调函数且不想编写大量代码时，使用匿名（ anonymous）内部类比较便捷。

内部类的分类：

根据内部类定义位置的不同，可以将内部类分为以下几种情况：

  1. 成员内部类
  2. 静态内部类
  3. 局部内部类
  4. 匿名内部类

注意：

  1. java中不允许非静态内部类定义静态变量和静态方法。
  2. java中允许非静态内部类定义静态常量。
  3. 一般情况下，内部类都会用private关键字进行修饰，不对外部开放。

成员内部类

成员内部类对象是外部类对象的一个成员。成员内部类不可以定义静态变量和静态方法，只可以定义静态常量。

成员内部类怎么访问外部类：

  1. 直接访问，并且可以访问外部类的私有成员。

静态内部类

静态内部类对象是外部类的成员。静态内部类可以定义静态变量、静态方法和静态常量。

静态内部类怎么访问外部类：

  1. 可以直接访问外部类私有的成员,就像定义在自己里面一样。
  2. 但是只能访问外部类静态成员（内部类有点类似于外部类的静态方法，只能访问外部类的静态成员）。

局部内部类怎么访问外部类：

  1. 直接访问，并且可以访问外部类的私有成员。
  2. 局部内部类访问自身所在外部类的方法中的局部变量必须是最终量（final）或者是实际的最终量（effectively final）。

匿名内部类

  • 概述：是内部类的简化写法。
  • 前提：必须先存在一个类或者一个接口（该类可以是抽象类，也可以是具体类）。
  • 格式：new 类名或者接口名() + {重写方法;}（+前面是创建一个对象，+后面是定义一个匿名类）。
  • 本质：创建了一个继承该类或者实现该接口的子类的对象（匿名类的本质是一个对象）。
  • 怎么理解：我要创建一个类或者接口的对象，但是我又不想重新写一个类去继承该类或者实现该接口。那我就可以在后面直接重写该类或者该接口的方法。
