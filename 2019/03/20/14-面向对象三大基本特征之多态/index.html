多态的概述

多态就是一个对象在不同时刻表现出不同状态。例如：猫可以是猫的类型（Cat cat = new Cat();），这时候猫可以表现出猫的状态；同时猫也可以只是动物的类型（Ainmal cat = new Cat();），这时候猫只可以表现出动物的状态。

理解多态的关键在于：

  • 一个对象可以在不同时刻表现出不同状态（该对象能被几个引用数据类型接受，它就有几种状态）。
  • 一个对象应该在何种时刻表现出何种状态（该对象被哪种数据类型接受就表现出哪种状态）。

多态的前提

  • 存在继承关系。
  • 子类重写父类的方法。
  • 父类引用指向子类的对象(如果父类引用不是指向子类对象，就会报ClassCastException)。

多态的好处

  • 提高了程序的维护性(由继承保证)。
  • 提高了程序的扩展性(由多态保证)。

例如：程序中有数据库连接的代码，且现在用的数据库是Oracle，可以通过定义一个父类Connection，再定义一个子类OracleConnection。

然后借助多态：

  1. 让子类OracleConnection继承父类Connection。
  2. 让子类OracleConnection重写父类Connection的方法。
  3. 让父类引用指向子类的对象（Connection connection = new OracleConnection();）。

接下来只要在程序中通过connection来调用各种方法就行了。如果将来换了数据库，假设把Oracle换成了Mysql，只需重新再定义一个子类MysqlConnection。

然后借助多态：

  1. 让子类MysqlConnection继承父类Connection。
  2. 让子类MysqlConnection重写父类Connection的方法。
  3. 让父类引用指向子类的对象（Connection connection = new MysqlConnection();）。

这样其他任何与调用数据库相关的代码就都不用再修改了，因为那些地方都是通过connection来调用各种数据库方法的。

多态的弊端

按照多态的成员访问特点（编译看左边），不能访问子类特有功能，因为如果是父类没有的功能，在编译的时候就会报错。如果想访问子类的特有功能，必须先向下转型，然后通过子类引用去访问。

多态中的转型问题

  • 向上转型(族谱)：父类引用指向子类对象(默认可以向上转型)。
  • 向下转型：父类引用转换子类引用(父类引用指向的是子类的对象)。
