异常概述

异常就是Java程序运行过程中出现的错误（通常有不同严重程度）。

异常由来：异常也是现实生活中的一个具体事务，也可以通过Java形式对其进行描述，并封装成对象。其实就是Java对不正常情况进行描述后的对象体现。Java将异常封装成了一个对象。

异常的分类

在Java程序设计语言中，异常对象都是派生于Throwable类的一个实例（即所有的异常类都继承于Throwable类）。如果Java中内置的异常类不能够满足需求，用户还可以创建自己的异常类。Java异常层次结构的简化示意图如下：

需要注意的是，所有的异常都是由Throwable继承而来，但在下一层立即分解为两个分支：Error和Exception。然后在下一层Exception又被分解为两个分支：编译期异常（受检查异常）和运行时异常即RuntimeException（非受检查异常）。

所以，大致上可以把异常分为三类：

  1. ERROR：

    • Error类层次结构描述了Java运行时系统的内部错误和资源耗尽错误。应用程序不应该抛出这种类型的对象，通常都是由虚拟机抛出的问题。如果出现了这样的内部错误，除了通告给用户，并尽力使程序安全地终止之外，再也无能为力了。这种情况很少出现。

    • 例如：堆栈溢出异常（StackOverflowError）、内存溢出异常（OutOfMemoryError）等。

  2. 编译期异常：

    • 除了RuntimeException及其子类，Exception中所有的子类都是，这种异常必须要处理，否则编译无法通过，推荐用try…catch进行处理。

    • 例如：打印输出异常（PrintException）等。

  3. 运行时异常（RuntimeError）：

    • RuntimeException及其子类都是，这种异常不处理编译也会通过，不过这样的程序会有安全隐患，在运行时可能出错，遇到这种异常不推荐用try…catch进行处理，一般需要通过修改代码的逻辑来解决。

    • 例如：空指针异常（NullPointerException）、数组越界异常（ArrayIndexOutOfBoundsException）、类型转换异常（ClassCastException）等。

JVM对异常的默认处理

如果我们没有对程序中的异常做出处理，JVM会对异常做出默认处理，处理流程如下：

  1. jvm发现了程序中的问题（例如：在程序中执行了1/0），java将这种常见的问题进行描述，并封装成对象（例如：1/0会被封装成一个ArithmeticException的对象）。
  2. 当问题发生后，jvm将该问题打包成了一个异常对象。
  3. jvm将异常对象抛给调用者main函数，（例如：1/0的异常对象是new ArithmeticException(“/by zero”)）。
  4. main函数收到这个问题时，有两种处理方式：一是可以自己将该问题处理，然后继续运行；二是如果自己没有针对的处理方式，只有交给调用main的jvm来处理。
  5. jvm有一个默认的异常处理机制：首先在控制台打印出在哪个线程中发生了异常，异常的类型，异常的信息，异常的调用栈；然后终止程序。

finally 子句

当代码抛出一个异常时，就会终止方法中剩余代码的处理，并退出这个方法的执行。如果方法获得了一些本地资源，并且只有这个方法自己知道，又如果这些资源在退出方法之前必须被回收，那么就会产生资源回收问题。一种解决方案是捕获并重新抛出所有的异常。但是，这种解决方案比较乏味，这是因为需要在两个地方清除所分配的资源。一个在正常的代码中；另一个在异常代码中。Java有一种更好的解决方案，这就是finally子句。

不管是否有异常被捕获，finally子句中的代码都被执行。事实上，我们认为在需要关闭资源时，用这种方式使用finally子句是一种不错的选择。

异常的处理原则

  1. 自己处理：
    • 如果异常在该方法能够处理，那就自己处理（及早处理）。
    • 如果异常后面的代码还要执行，只能自己处理，因为如果抛给上级处理，该方法一定会处于某个上级的try语句块中，一旦上级捕获并处理完异常，就不会再回去执行try语句块中未执行的代码，这样异常后面的代码自然就无法被执行了。
  2. 抛给上级处理：
    • 如果自己处理不了就往上抛。
    • 如果异常后面的代码不需要执行，就可以往上抛。

异常的注意事项

  1. 子类重写父类方法时，子类的方法必须抛出相同的异常或父类异常的子类。

  2. 如果父类抛出了多个异常，子类重写父类时，只能抛出相同的异常或者是他的子集，子类不能抛出父类没有的异常。

  3. 如果被重写的方法没有异常抛出，那么子类的方法绝对不可以抛出异常，如果子类方法内有异常发生，那么子类只能try，不能throws。
