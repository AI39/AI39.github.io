泛型概述

从Java程序设计语言1.0版发布以来，变化最大的部分就是泛型。

泛型正是我们需要的程序设计手段。使用泛型机制编写的程序代码要比那些杂乱地使用Object变量，然后再进行强制类型转换的代码具有更好的安全性和可读性。泛型对于集合类尤其有用。

泛型程序设计（Generic programming）意味着编写的代码可以被很多不同类型的对象所重用。

泛型的好处

  1. 提高了程序的安全性。
  2. 将运行期遇到的问题转移到了编译期。
  3. 省去了类型强转的麻烦。

泛型的应用

泛型类

把泛型定义在类上就是泛型类。一个泛型类（generic class）就是具有一个或多个类型变量的类。泛型类定义中的泛型类型指定方法的返回类型以及域和局部变量的类型。用具体的类型替换泛型类型就可以实例化泛型类型。

格式：public class 类名<泛型类型1, …>

注意:

  1. 泛型类型必须是引用类型。
  2. 泛型类型的命名规则只要符合标识符的限定即可，但是一般情况是用单个的大写字母标识，例如：用T, U, K来代表泛型类型。
  3. 如果没有在实例化的时候通过<>操作符用具体的类型替换泛型类型，就会默认用Object类型替换泛型类型。

泛型方法

把泛型定义在方法上就是泛型方法。可以定义一个带有泛型类型的简单方法。当调用一个泛型方法时，在方法名前的尖括号中放入具体的类型。泛型方法定义中的泛型类型指定方法的返回类型以及局部变量的类型。用具体的类型替换泛型类型就可以调用泛型方法。

格式：public <泛型类型> 返回类型 方法名(泛型类型1, …)

注意：

  1. 泛型类型一样必须是引用类型。
  2. 泛型类型的命名规则只要符合标识符的限定即可，但是一般情况是用单个的大写字母标识，例如：用T, U, K来代表泛型类型。
  3. 如果没有在调用方法的时候通过<>操作符用具体的类型替换泛型类型，就会默认用Object类型替换泛型类型。
  4. 泛型类型放在修饰符的后面，返回类型的前面。
  5. 泛型方法可以定义在普通类中，也可以定义在泛型类中。

泛型接口

把泛型定义在接口上就是泛型接口。泛型接口定义中的泛型类型指定实现该接口的子类的方法的返回类型以及域和局部变量的类型。用具体的类型替换泛型类型就可以定义实现了泛型接口的类。

格式：public interface 接口名<泛型类型1…>

注意:

  1. 泛型类型必须是引用类型。
  2. 泛型类型的命名规则只要符合标识符的限定即可，但是一般情况是用单个的大写字母标识，例如：用T, U, K来代表泛型类型。
  3. 如果没有在实例化的时候通过<>操作符用具体的类型替换泛型类型，就会默认用Object类型替换泛型类型。
  4. 实现泛型接口的类可以不是泛型类，也可以是泛型类。

泛型高级(通配符)

固定的泛型类型系统使用起来并没有那么令人愉快，Java的设计者发明了一种巧妙的（仍然是安全的）“解决方案”：通配符类型。

通配符类型中，允许类型参数变化。

通配符类型分为三种：

  1. <?>：任意类型，如果没有明确，那么就是Object及其任意的Java类了。
  2. <? extends E>：向下限定，E及其子类。
  3. <? super E>：向上限定，E及其父类。
