多线程概述

多线程（multithreading），是指从软件或者硬件上实现多个线程并发执行的技术。

在计算机编程中，一个基本的概念就是同时对多个任务加以控制。许多程序设计问题都要求程序能够停下手头的工作，改为处理其他一些问题，再返回主进程。

多线程是为了同步完成多项任务，不是为了提高运行效率，而是为了提高资源使用效率来提高系统的效率。线程是在同一时间需要完成多项任务的时候实现的。

Java对多线程的支持是非常强大的，他屏蔽掉了许多的技术细节，让我们可以轻松的开发多线程的应用程序。

进程

  1. 进程就是正在运行的程序，是系统进行资源分配和调用的独立单位。
  2. 每一个进程都有它自己的内存空间和系统资源。

线程

  1. 在一个进程内部可以执行多个任务，而这每一个任务我们就可以看成是一个线程。
  2. 线程是进程中的单个顺序控制流，是一条执行路径。
  3. 一个进程如果只有一条执行路径，则称为单线程程序。
  4. 一个进程如果有多条执行路径，则称为多线程程序。

进程与线程的区别

  • 进程与线程本质的区别在于每个进程拥有自己的一整套变量，而线程则共享数据。

线程的调度模型

线程调度有两种常见模型：分时调度模型和抢占式调度模型。Java使用的是抢占式调度模型。

分时调度模型

平均分配，各线程轮流获取时间片来执行各自的流程。

抢占式调度模型

按优先级分配，线程的优先级越高，越容易获取时间片来执行自己的流程。

Java程序的运行原理

  1. Java命令会启动JVM，即启动了一个进程（JVM就是一个进程）。
  2. 该进程会启动一个主线程，然后主线程调用某个类的main方法，所以main方法都是运行在主线程里。

注意：

  1. 每个Java进程会分配一jvm个实例。
  2. JVM是多线程的，至少会创建一个main线程和一个GC线程（垃圾回收）。

Thread类

Java中创建线程有两种方法：

  1. 一种方法是定义一个Thread类的子类，在子类中重写Thread类的run()方法，再创建继承Thread类的子类的对象，并启动线程。
  2. 另一种方法是定义一个子类实现Runnable接口，在子类中重写Runnable接口的run()方法，创建实现Runnable接口的子类的对象，再创建一个Thread类的对象，并将Runnable接口的子类对象作为一个参数来传递，然后启动线程。

在上面两种创建线程的方法中，第一种方法定义了一个Thread类的子类，所以该子类自然就继承了很多Thread类中的方法，而创建的Thread类的子类的对象就可以调用这些方法；第二种方法创建了一个Thread类的对象，并将Runnable接口的子类的对象作为一个参数来传递进去，而创建的Thread类的对象也可以调用Thread类里面提供的方法。

线程安全问题

在大多数实际的多线程应用中，两个或两个以上的线程需要共享同一数据的存取。如果两个线程存取相同的对象，并且每一个线程都调用了一个修改该对象状态的方法，可能会产生讹误的对象。

在上面的例子中，创建了两个线程：count++线程0和count++线程1。让这两个线程分别去执行同一个任务：将共享的（即同一个）count变量从1加到100，并打印输出。理想的输出结果应该是count从0到100连续无重复的输出，但是现实的输出结果却可能是出现遗漏且重复的输出。

造成这个问题的主要原因在于多个线程不停地来回切换执行：当第一条线程在执行count++语句后还没来得及执行打印输出语句，这时候第二个线程抢到了时间片，也执行了count++语句，可能这时候第一条线程又抢到了时间片，执行了打印输出语句，然后第二条线程抢到了时间片，执行了打印输出语句，所以就出现了打印结果出现遗漏且重复。

引起线程安全问题的原因

  1. 多线程。
  2. 共享数据。
  3. 操作共享数据并非原子操作。

解决线程安全问题的办法

将引起线程安全问题的原因或条件破坏掉就可以解决线程安全问题，但是多线程和共享数据这两个条件无法破坏，所以只能将物理上非原子的操作共享数据改成逻辑上原子的操作共享数据。

所谓原子操作是指不会被线程调度机制打断的操作；这种操作一旦开始，就一直运行到结束，中间不会有任何context switch（切换到另一个线程）。

将物理上非原子的操作共享数据改成逻辑上原子的操作共享数据的方法有：

  1. 锁对象。
  2. 同步方法。
  3. 同步代码块。

同步的优缺点

上面解决多线程安全问题的方法，从本质上讲都是使用锁对象，将物理上非原子的操作共享数据改成逻辑上原子的操作共享数据。

多个线程，使用同一个锁对象，即构成同步。

同步的优点：

  • 解决了多线程的安全问题。

同步的缺点：

  1. 消耗资源（当线程很多时，每个线程运行的时候都需要去判断同步锁，这是很消耗系统资源的），影响效率。
  2. 如果出现了嵌套锁，容易产生死锁。

线程间通信

用多线程去操作共享数据可以提高系统效率（并不是运行效率），但是多线程共享数据会出现线程安全问题，为了解决线程安全问题，可以对数据操作进行加锁，但是对数据加锁又会带来死锁问题。因此，为了解决死锁问题，线程之间应该相互配合，这就涉及到了线程间通信。

线程池

构建一个新的线程是有一定代价的，因为涉及与操作系统的交互。如果程序中创建了大量的生命期很短的线程，应该使用线程池（thread pool）。一个线程池中包含许多准备运行的空闲线程。将Runnable对象交给线程池，就会有一个线程调用run()方法。当run方法退出时，线程不会死亡，而是在池中准备为下一个请求提供服务。

另一个使用线程池的理由是减少并发线程的数目。创建大量线程会大大降低性能甚至使虚拟机崩溃。如果有一个会创建许多线程的算法，应该使用一个线程数“固定的”线程池以限制并发线程的总数。

Java从JDK5之后开始支持线程池。

定时器Timer

定时器是一个应用十分广泛的线程工具，可用于调度多个定时任务以后台线程的方式来执行。可安排任务执行一次，或者定期重复执行。

在Java中，可以通过Timer类和TimerTask类来实现定义调度功能。

具体流程如下：

  1. 定义一个子类继承TimerTask类，在子类中重写run()方法。
  2. 创建一个Timer类对象，再创建一个继承TimerTask类的子类对象，并将Timer类对象作为一个参数传入。
  3. 调用Timer类对象的schedule()方法，并将继承TimerTask类的子类对象作为一个参数传入，即可定时执行该任务。
