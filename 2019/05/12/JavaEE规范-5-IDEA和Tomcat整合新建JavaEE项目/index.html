5-IDEA和Tomcat整合新建JavaEE项目

发表于 2019-05-12 | 分类于 做一个有骨气的Coder ， 我的JavaEE哪有这么熟练 ， JavaEE规范

JavaEE工程的创建

  1. 打开软件，点击“Create New Project”。
  2. 依次选择相应的“Project SDK”、“Java EE version”和“Application Server”（例如：在下图中“Project SDK”选择的是“JDK1.8”、“Java EE version”选择的是“Java EE 8”，“Application Server”选择的是“Tomcat 8.5.37”）,并选中“Web Application”，表示创建的是JavaWeb项目，然后点击“Next”。
  3. 在上面输入新创建工程的名字并更改工程所在目录，在底下输入新创建工程下新建模块的名字并更改模块所在目录，然后点击“Finish”。
  4. 项目创建成功后，会进入如下界面。

JavaEE项目的配置

  1. 选中新创建的项目或者项目下的任意一个模块，点击右键，选中“Open Module Settings”。
  2. 点击“Open Module Settings”后，可以看到如下的设置界面。
  3. 在设置界面里共有五个配置项，分别为：Project、Modules、Libraries、Facets和Artifacts，下面将分别介绍这5个配置项。

Project配置项

选中Project配置项可以进入如下界面。

在Project配置项下，可以配置如下信息：Project name、Project SDK、Project language level和Project compiler output。

Project name

定义工程的名称（一个Project里有多个module，每个module都可以是一个单独的应用）。

Project SDK

设置该项目使用的JDK，也可以在此处新添加其他版本的JDK。

Project language level

这个和Project SDK类似。区别在于，假如你设置了JDK1.8，却只用到1.6的特性，那么这里可以设置语言等级为1.6，这个是限定项目编译检查时最低要求的JDK特性。

Project compiler output

项目中的默认编译输出目录，实际上每个模块可以自己设置特殊的输出目录（Modules - (project) - Paths - Use module compile output path），所以这个设置有点鸡肋。

Modules配置项

选中Modules配置项可以进入如下界面。

在Modules配置项下，可以配置如下信息：增删模块和模块配置。

增删模块

一个项目中可以有多个子项目），每个子项目相当于一个模块。一般我们的项目只有单独的一个模块，IDEA默认也是单子项目的形式，所以只需要配置一个模块。

可以通过左边的“+-”来实现模块的增加与删除。

模块配置

每个模块都对应了Sources、Path和Dependencies三大配置选项。

  • Paths：可以指定项目的编译输出目录，即项目类和测试类的编译输出地址（替换掉了Project的默认输出地址）。
  • Dependencies：项目的依赖。

模块配置的Sources配置项

选中模块配置的Sources配置项可以进入如下界面：

这里显示项目的目录资源，不同颜色代表不同的类型。

模块配置的Path配置项：

选中模块配置的Path配置项可以进入如下界面：

这里可以指定模块（即子项目）的编译输出目录，即项目类和测试类的编译输出地址，指定以后将替换掉Project的默认输出地址。我们可以更改这里的配置，使得编译后的文件能够放置在正确的目录。

模块配置的Dependencies配置项：

选中模块配置的Dependencies配置项可以进入如下界面：

这里可以添加、删除和编辑当前模块的依赖。在导包的过程中往往要对这里进行相关的配置。

Libraries配置项

选中Libraries配置项可以进入如下界面。

这里是project级别的依赖库，这里的库可以被其他的module引用。

Facets配置项

选中Facets配置项可以进入如下界面。

在Facets配置项下，可以配置如下信息：Name、Deployment Ddscriptions和Web Resource Directories。

Name

Name通常不需要修改，默认是Web。

Deployment Ddscriptions

这里Facets的部署文件描述符可以帮助我们的项目自动生成WEB-INF目录以及里面的web.xml。

Web Resource Directories。

这里的web资源目录用于指定我们web项目的那个目录，在这个目录里存放我们的web资源，该目录通常是web目录。部署的时候IDEA会把该目录下的所有内容复制到对应的out目录文件夹里。

Artifacts配置项

选中Artifacts配置项可以进入如下界面。

这里配置的是项目的编译结果和各种资源文件，怎么组织在一起，最终打包发布。某个模块有了 Artifacts就可以部署到应用服务器中了。

IDEA和Tomcat整合新建JavaEE项目所需要的配置

开发web应用时，不同类型的文件有严格的存放规则，否则不仅可能会使web应用无法访问，还会导致web服务器启动报错。

用Tomcat开发web应用时web应用的组成结构如下图所示：

用IDEA新创建一个项目时，默认的文件结构及配置如下图所示：

可以看到在模块目录下有两个平级的目录：

  1. src：在src中存放的是Java的源代码，包括Servlet的Java源代码等。
  2. web：每个项目下的模块的web目录即对应用Tomcat开发web应用时的web应用目录。我们开发的web应用的文件结构必须严格遵守用Tomcat开发web应用时web应用的组成结构图。

为了开发的方便，我们需要做出如下配置：

  1. 更改指定模块的编译输出目录。
  2. 配置Tomcat。

配置Tomcat

  1. 点击“Toolbar”中的配置Tomcat按钮，然后点击“Eidt Configurations”选项，进入Tomcat配置界面。
  2. 在Tomcat配置界面中，主要配置如下配置选项：Server和Deployment。
  3. 配置Server配置选项时，可以修改Tomcat服务器运行的端口号，一般可设置为80，这样从浏览器访问的时候就可以省略端口号。
  4. 配置Deployment配置选项时，可以增删要部署的应用。
  5. 配置Deployment配置选项时，还可以为要部署的应用设置虚拟映射，使得要部署的应用被关联到该Tomcat的webapps目录下（例如：下面的例子中，在浏览器输入域名:端口号/practice_0/资源名即可访问practice_0模块所对应的应用下的资源）。

更改指定模块的编译输出目录

我们可以对照用Tomcat开发web应用时web应用的组成结构图，在web目录下的WEB-INF目录下新建一个classes目录，用来存放相应的web应用资源。

我们还可以通过配置Modules配置项中的Path配置项，来更改指定模块（即子项目）的编译输出目录，这里一般直接把编译输出目录改为web目录下的WEB-INF目录下的classes目录，这样在src目录下编写的java源文件，例如Servlet的Java源代码在编译后生成的字节码文件就会被直接放置到web目录下的WEB-INF目录下的classes目录，而不再需要我们手动放置了。

具体配置过程如下：

  1. 选中新创建的项目或者项目下的任意一个模块，点击右键，选中“Open Module Settings”，进入配置界面。
  2. 在设置界面里选中Modules配置项进入模块配置界面，再点击Path配置项进入Path配置界面。
  3. 选中“Use module compile output path”，将“Output path”修改为web目录下的WEB-INF目录下的classes目录，“Test output path”可以不做修改，具体如下图所示：

导包

我们可以对照用Tomcat开发web应用时web应用的组成结构图，在web目录下的WEB-INF目录下新建一个lib目录，用来存放Java类运行时所需要的jar包。

这里给出一个例子，导入两个与处理文件上传有关的组件，具体的导包过程如下：

  1. 先从官网下载相关的组件（即要导入的包），并放置到lib目录下。
  2. 同时选中lib目录下的所有包，点击右键，选择“Add as Library…”选项。
  3. 默认选择包的有效作用域为整个项目，这样同一个项目下的其他模块如果也需要使用这些包就不用再做相同的操作了。
  4. 在设置界面里选中Modules配置项进入模块配置界面，再点击Dependencies配置项进入Dependencies配置界面，可以在该界面下看到已经建立了相应的依赖关系。
  5. 一般在同一个项目下的其他模块如果也导入了相同的包，在Dependencies配置界面可能需要手动添加相应的依赖关系。点击“+”选择“Library”，再选择要建立的依赖关系。

IDEA运行Tomcat时的执行流程

在IDEA中的JavaEE项目，点击Run或Debug按钮就可以运行Tomcat，IDEA运行Tomcat时的执行流程如下：

  1. 编译，IDEA在保存/自动保存后不会做编译，不像Eclipse的保存即编译，因此在运行server前会做一次编译。编译后的class文件存放在指定的项目编译输出目录下。
  2. 根据Artifacts中的设定对目录结构进行创建。
  3. 拷贝web资源的根目录下的所有文件到artifact的目录下。
  4. 拷贝编译输出目录下的classes目录到artifact下的WEB-INF下。
  5. 拷贝lib目录下所需的jar包到artifact下的WEB_INF下。
  6. 运行server，运行成功后，如有需要，会自动打开浏览器访问指定URL。

SE项目和EE项目新建文件时区别

当采用相对路径的写法新建文件时，SE项目和EE项目新建的文件最后所在的实际路径有很大区别，原因在于SE项目和EE项目相对路径相对的对象不同。

JDK里面的相对路径，指的是相对java虚拟机调用的目录。在哪个目录下调用jvm，就相对的是哪个路径。
