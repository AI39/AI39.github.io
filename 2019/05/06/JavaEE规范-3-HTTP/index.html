HTTP简介

web浏览器与web服务器之间的一问一答的交互过程必须遵循一定的规则，而这个规则就是HTTP协议。

HTTP是HyperText Transfer Protocol(超文本传输协议)的简写，它是TCP/IP协议族的一个应用层协议，用于定义web浏览器与web服务器之间交换数据的过程及数据本身的格式。

HTTP协议是学习web开发的基石，不深入了解HTTP协议，就不能说掌握了web开发，更无法管理和维护一些复杂的web站点。

HTTP协议的版本

HTTP协议主要有HTTP1.0和HTTP1.1。

HTTP/1.0

基于HTTP协议的客户机/服务器模式的信息交换过程，如下图所示：

它分为四个过程：建立连接、发送请求消息、发送响应消息、关闭连接。

HTTP1.0的特点

  1. 浏览器与web服务器的连接过程是短暂的，每次连接只处理一个请求和响应。对每一个页面的访问，浏览器与web服务器都要建立一次单独的连接。
  2. 浏览器到WEB服务器之间的所有通讯都是完全独立分开的请求和响应。
  3. 是无状态连接。

HTTP1.1的特点

  1. 在一个TCP连接上可以传送多个HTTP请求和响应。
  2. 多个请求和响应过程可以重叠。
  3. 增加了更多的请求头和响应头，比如Host、If-Unmodified-Since请求头等。

一次完整的HTTP事务

一次完整的HTTP事务包含如下几个过程：

  1. 域名解析。
  2. 发起TCP的三次握手。
  3. 连接成功后，发起HTTP请求。
  4. 服务器响应HTTP请求。
  5. 浏览器解析HTML代码，并请求HTML代码中的资源（如JS、CSS、图片等）。
  6. 浏览器对页面进行渲染呈现给用户。

HTTP事务

HTTP事务 = 请求命令 + 响应结果

请求行

请求行的格式如下：请求方式 资源路径 HTTP版本号。例如：GET /temp3o116.shtml HTTP/1.1。

请求方式如下：GET、POST、HEAD、OPTIONS、DELETE、PUT。

用户如没有设置，默认情况下浏览器向服务器发送的都是GET请求，例如在浏览器直接输地址访问，点超链接访问等都是GET，用户如想把请求方式改为POST，可通过更改表单的提交方式实现。

不管POST或GET，都用于向服务器请求某个web资源，这两种方式的区别主要表现在数据传递上。

GET方式

如请求方式为GET方式，则可以在请求的URL地址后以?的形式带上交给服务器的数据，多个数据之间以&进行分隔，例如：
GET /mail/1.html?name=abc&password=xyz HTTP/1.1

GET方式的特点：在URL地址后附带的参数是有限制的，其数据容量通常不能超过1K。

POST方式

如请求方式为POST方式，则可以在请求的正文内容中向服务器发送数据。

Post方式的特点：传送的数据量无限制。

请求头

请求头字段用于客户端在请求消息中向服务器传递附加信息，主要包括客户端可以接受的数据类型(MIME类型)、压缩方法、语言以及发出请求的超链接所属页面的URL地址等信息。

使用消息头，可以实现HTTP客户机与服务器之间的条件请求和应答，消息头相当于服务器和浏览器之间的一些暗号指令。

每个消息头包含一个头字段名称，然后依次是冒号、空格、值、回车和换行符。

常用的请求头如下表所示：

请求头 作用
Accept 浏览器可接受的MIME类型
Accept-Charset 浏览器通过这个请求头告诉服务器，它支持哪种字符集
Accept-Encoding 浏览器能够进行解码的数据编码方式，比如gzip
Accept-Language 浏览器所希望的语言种类，当服务器能够提供一种以上的语言版本时要用到。可以在浏览器中进行设置
Host 初始URL中的主机和端口
Referer 包含一个URL，用户从该URL代表的页面出发访问当前请求的页面（防盗链）
Content-Type 内容类型
If-Modified-Since 服务器利用这个头与服务器的文件进行比对，如果一致，则告诉浏览器从缓存中直接读取文件
User-Agent 浏览器类型
Content-Length 表示请求消息正文的长度
Connection 表示是否需要持久连接。如果服务器看到这里的值为“Keep -Alive”，或者看到请求使用的是HTTP 1.1（HTTP 1.1默认进行持久连接），表示需要持久连接
Cookie 这是最重要的请求头信息之一
Date 请求时间

注意：

  1. 请求头字段名是不区分大小写的，但习惯上讲每个单词的第一个字母大写。
  2. 整个请求头部分中的各行消息头可按任何顺序排列。
  3. 许多请求头字段都允许客户端在值部分指定多个可接受的选项，多个选项之间以逗号分隔。
  4. 有些头字段可以出现多次。
  5. MIME类型针对html文档而言是text/html；针对图片而言是image/png、image/gif等；针对音频资源是audio/mp3等。

响应行

响应行的格式如下：HTTP版本号　状态码　原因叙述。例如：HTTP 1.1 200 OK。

状态码用于表示服务器对请求的各种不同处理结果和状态，它是一个三位的十进制数。响应状态码分为5类，使用最高位为1到5来进行分类。

常见的状态码如下表所示：

请求头 作用
200~299 表示成功接收请求并已完成整个处理过程。例如：200表示一切正常，返回的是正常请求结果
300~399 （重定向）为完成请求，客户需进一步细化请求。例如：301、302、307表示临时重定向，指出被请求的文档已被临时移到别处，此文档的新的URL在Location响应头中给出
400~499 客户端的请求有错误。例如：404表示找不到，服务器上不存在客户机所请求的资源
500~599 服务器端出现错误。例如：500表示服务端的程序发生错误

响应头

响应头字段用于向客户端传递附加信息。

常用的响应头如下表所示：

响应头 作用
Location 指示新的资源的位置
Server 指示服务器的类型。例如：apache tomcat
Content-Encoding 服务器发送的数据采用的编码类型。例如：gzip
Content-Length 告诉浏览器正文的长度。
Content-Language 服务发送的文本的语言。例如：zh-cn
Content-Type 服务器发送的内容的MIME类型。例如：text/html
Last-Modified 文件的最后修改时间
Refresh 指示客户端刷新频率，单位是秒
Content-Disposition 指示客户端保存文件
Set-Cookie 服务端将生成的cookie发送给浏览器端，浏览器再次访问服务器时，会带上cookie请求头
Expires 声明了一个网页或URL地址不再被浏览器缓存的时间，一旦超过了这个时间，浏览器都应该联系原始服务器
Cache-Control 指定所有缓存机制
Connection 表示是否需要持久连接
Date 响应时间

注意：

  1. 浏览器看到Content-Type这个响应头，就会把当前的资源当做content-type响应头里面的内容类型来解析。如果响应头Content-Type设置的值是text/html，而返回的响应是图片资源，则图片可能会无法正常加载。
